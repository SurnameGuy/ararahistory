Ararahistory: Como foi fazer

Certo, agora que estamos introduzidos, vamos ao que realmente importa: como foi fazer o site.


Bem, fazer o site foi um processo interessante, para dizer o mínimo. Alguns de nós já tinhamos experiência com criação de sites e tudo mais. Mas já que estamos aqui para aprender, por que não tentar coisas novas, não é?

A primeira versão do site foi feita com o báscico do básico da prog web: HTML, CSS e JS. Foi uma base sólida e até bem funcional, modéstia à parte. O ponto é que a equipe decidiu fazer uma mudança, sair do HTML e ir para o React, não só por ser uma coisa que ainda não conhecíamos mas que poderíamos aprender ainda mais se o projeto fosse feito nele.


Após migrarmos para o novo ambiente, enfrentamos as dificuldades de sempre: uma linguagem nova e quase desconhecida, funcionalidades diferentes mas parecidas, novas praticidades e complexidades. O de sempre.

Foi uma experiência, sem sombra de dúvidas. Cada um de nós estava cuidando de diferentes áreas do site, conteúdo, estrutura, funcionalidades, etc. Mas ao final, voltamos ao início, retornando ao HTML, agora com Django. E o que está vendo agora é o resultado disso, dessa terceira versão do projeto, depois de sair do puro HTML, para React, para voltar ao HTML. É, o fruto não cai muito longe do pé... ou algo assim.

O que aprendemos?

No fim, esse curso ensinou muita coisa para nós. Desde o básico da prog web até coisas que nenhum de nós conhecia. Aqui está um pouco do que vimos:

HTML

A linguagem de marcação de hipertexto, o HTML é o mais básico para se saber quando se está criando qualquer coisa para a web. Como é apenas uma forma de explicar para o navegador o que vai em cada lugar, não se pode dizer que é uma linguagem de programação.

CSS

A "Folha de Estilos em Cascata" ou Cascading Style Sheets, o CSS serve para estilizar os elementos no HTML. De mudanças de cor até rotação, espaçamento e quase tudo mais que se possa pensar, o CSS é capaz de fazer com os elementos dentro do site.
